Notes on several of the problems:

  • Problem B - The Drunk Jailer: This problem has a nice analytical solution. If there are n cells, then exactly floor( sqrt(n) ) prisoners escape. I am quite embarassed to admit that I wrote this problem, though I did not invent it, and I didn't realize it had an analytic solution until three days after the contest. See if you can derive the solution.
  • Problem C - Lingo: Most of the incorrect submissions were either missing the initial newline, or they forgot to read the initial newline in the dataset(s). If the first line of the output is wrong, the problem is marked as wrong.
  • Problem D - Number Base Conversion: Java does support arbitratry bases, but only up to base 36 (numbers and one set of letters).
  • All problems: All the judges' solutions were written in either C or C++ (strangely enough, none of the judges' solutions used Java). None of the judges' solutions used template libraries or "String" objects.
